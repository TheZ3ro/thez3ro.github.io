How I made my Hidden Service

Oct 4, 2014 • Category: En

Setting up an Hidden Service isn't so Hard.

You can see the Tor Documentation,
but I want to share some tool that I have used.

The Hidden Service is online at this address: http://thezeromj2umfc42.onion/
The Clear Website is online at this address: http://thezero.org

Both the sites are made with Jekyll and the Theme used is Lambda.
The Clear one is hosted on Github Pages.
I use git to update both the sites, this is the git repository.
There are 2 branch, master (where there is the site) and source (where there are the Jekyll site's source)

Jekyll-Git is a (very) simple tool that I wrote for Building Jekill sites from source and Deploy it directly on Github (in a repository with master and source branches).

The HS (Hidden Service) is hosted on a VPS at YourServer.se, in Latvia, because their ToS (Term of Service) allow hosting a Tor node.
Here is a list of VPS that allow Tor.
You can also set up an HS from your PC or RaspberryPi.

In the Jekyll-Git repository is also available a config/shedule.rb file for the whenever ruby gem, so I can do a "git pull" on the VPS from cron.

In the server is obviously installed Tor.
I also use my thc for manage the Hidden Services installed.

Apache and LightHTTPd are not recommended as webserver cause they can leak the server's IP address or log the visitors' IPs.
I prefer using my fork of Nano-server, a simple webserver written in node.js but you can also use the built-in Jekyll webserver

The .onion address was generated by Shallot (written in C for CPU) or scallion (written in C# for GPU).
